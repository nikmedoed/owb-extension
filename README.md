# OWB Tools

Набор инструментов для Ozon/WB: выгрузка карточек товаров (включая отзывы), мониторинг цен с историей и двунаправленная синхронизация с локальным сервером.

## Зачем проект

- Собирать карточку товара в текст, удобный для анализа нейросетями.
- Забрать сразу много карточек из всех вкладок окна браузера.
- Хранить историю цен локально без внешних зависимостей.
- При доступном сервере объединять данные с нескольких устройств в локальной сети.

## Что в репозитории

- `extension/` - Chrome extension (Manifest V3).
- `server/` - локальный HTTP API сервер синхронизации на Python + SQLite.

## Ключевая функциональность

### 1) Выгрузка карточек (Ozon/WB)

- На странице товара есть кнопки: `Скачать`, `без отзывов`, `все отзывы`, `в буфер`.
- Popup поддерживает батч-операции по всем маркетплейс-вкладкам текущего окна:
  - скачать карточки в файлы;
  - скачать карточки с максимумом отзывов;
  - скопировать объединенный текст в буфер;
  - скопировать объединенный текст с максимумом отзывов.
- Объединение текста в батче делается с разделителем `---`.
- Результат последней сессии сохраняется и показывается в popup (с кнопкой копирования).

### 2) Логика сбора на Ozon

Текущий сбор строится как последовательный проход:

1. Чтение шапки товара (название, бренд, цена, рейтинг, число отзывов).
2. Переход к блоку описания, попытка развернуть скрытый текст.
3. Чтение описания, включая longread-кейсы с изображениями.
4. Переход к характеристикам и чтение характеристик.
5. Переход к отзывам и последовательный сбор отзывов (по умолчанию до 100, либо больше в режиме `все отзывы`).
6. В итог добавляется строка со средней оценкой.

Для отзывов применяется остановка по отсутствию прогресса и дополнительный шаг прокрутки перед завершением.

### 3) Мониторинг цен

- История цен хранится в IndexedDB расширения:
  - DB: `owb-price-history-ext`
  - Stores: `intervals`, `products`
- Модель хранения интервальная:
  - если цена не изменилась, продлевается текущий интервал (`lastTs`);
  - если цена изменилась, создается новый интервал.
- В popup для активного товара рисуется быстрый график (текущая/мин/макс, диапазон дат).
- Есть сброс истории по текущему товару.

### 4) Локальный сервер и двунаправленный sync

- По умолчанию адрес сервера: `http://127.0.0.1:8765`.
- Режимы:
  - `local`: работа только в локальном хранилище расширения.
  - `sync`: пуш локальных изменений на сервер + пулл серверных изменений обратно.
- Автосинк запускается при старте расширения, при записи новых цен и при запросах данных.
- Если сервер недоступен или URL пустой, фактически используется локальный режим.
- Подтягивание серверных данных селективное:
  - для карточек в листингах: только минимумы (`/api/min-batch`);
  - для истории товара: только история конкретного `pidKey` (`/api/history`).
- С обеих сторон используется merge интервалов, чтобы не плодить дубли.

### 5) Страница настроек (Options)

- Управление sync (чекбокс + URL сервера + проверка доступности).
- Дефолтный URL подставляется автоматически.
- Экспорт/импорт всей БД истории цен.
- Блок диагностики БД без ручной кнопки:
  - всего записей;
  - товаров;
  - интервалов;
  - среднее интервалов на товар;
  - распределение Ozon/WB;
  - время последней активности;
  - статус.

## Быстрый старт

1. Открыть `chrome://extensions`.
2. Включить `Developer mode`.
3. Нажать `Load unpacked` и выбрать папку `extension`.
4. (Опционально) запустить сервер:
   - `cd server`
   - `python local_price_server.py`
5. В `Options` расширения включить `Синхронизация`, если нужен режим `sync`.

## API локального сервера

- `GET /ping` - доступность сервера.
- `POST /api/intervals/bulk` - прием батча интервалов цен.
- `GET /api/changes?since=<ts>&sinceId=<id>&limit=<n>` - инкрементальные изменения.
- `GET /api/history?pidKey=<pidKey>` - история по товару.
- `POST /api/min-batch` - минимальные цены по списку `pidKeys`.
- `POST /api/price` - режим одиночного наблюдения (служебный, совместимость).

## Важные технические детали

- `pidKey` формат: `ozon:<id>` или `wb:<id>`.
- Последняя батч-сессия хранится в `chrome.storage.local` по ключу `owb-last-extract-session`.
- При большом батч-результате текст сессии сохраняется с ограничением (защита от переполнения хранилища).
- Серверная БД: `server/price_history.sqlite` (можно переопределить env-переменной `PRICE_SERVER_DB`).

## Ограничения и текущий фокус

- DOM маркетплейсов часто меняется; селекторы и прокрутка требуют периодической актуализации.
- На длинных карточках с тяжелым контентом сбор отзывов может зависеть от скорости догрузки страницы.
- Архитектура уже рассчитана на multi-device sync, но для стабильности важно держать сервер доступным в локальной сети.
