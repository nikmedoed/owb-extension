# OWB Tools Workspace

## Структура

- `extension/` - Chrome extension (MV3), грузить в `chrome://extensions` как `Load unpacked`
- `server/` - локальный сервер синка цен (`local_price_server.py`, SQLite)
- `Marketplace Instant Exporter with Reviews-1.3.0.user.js` - исходный userscript-референс

## Важные артефакты по логике цен (актуально)

### 1) Где и как собираются цены

Сбор идёт в двух контурах:

- **Страница товара**:
  - Ozon: виджеты `webPrice`/`webSale`
  - WB: `priceBlock*` и соседние блоки цены
- **Карточки в листингах/поиске/корзиноподобных списках**:
  - Ozon: ссылки `a[href*="/product/"]`
  - WB: ссылки `a[href*="/catalog/"][href*="/detail"]`

Для карточек сначала берётся идентификатор товара из URL, затем цена ищется внутри контейнера карточки через эвристику `findPriceInCard`.

### 2) Где хранится история

Базово история хранится **на стороне расширения** (service worker IndexedDB):

- DB: `owb-price-history-ext`
- Store `intervals` — интервалы цен
- Store `products` — сводка по товару (хвост и минимум)

При включённом режиме `sync` расширение синхронизирует эти интервалы с локальным сервером (`server/local_price_server.py`).

Формат интервала:

- `pidKey` (`ozon:<id>` / `wb:<id>`)
- `price`, `currency`
- `firstTs` (когда впервые увидели цену)
- `lastTs` (когда в последний раз увидели ту же цену)

### 3) Алгоритм записи (без лишних дублей)

При новом наблюдении цены товара:

- если последняя запись этого товара имеет ту же цену и валюту → обновляется только `lastTs`
- если цена изменилась → создаётся новый интервал (`firstTs = lastTs = now`)

Итог: не хранится 50 одинаковых точек подряд, но временные границы цены сохраняются.

### 4) Отрисовка графика

На странице товара график рисуется в блоке под ценой:

- источник данных: интервалы товара из background DB
- интервалы преобразуются в точки (начало/конец интервала)
- показываются:
  - линия изменения
  - `Мин` / `Макс`
  - даты начала/конца диапазона
  - tooltip по наведению на точки

Если якорь цены временно не найден, блок уходит во floating-режим.

### 5) Минимальная цена на карточках

Для карточек товаров запрашивается батч минимумов (`pidKey[] -> min`) и поверх карточек рендерится бейдж:

- `Мин <цена>`

Это работает на страницах, где карточку можно связать с товаром по URL/ID.

### 6) Экспорт/импорт всей базы

Экспорт/импорт делается без обхода страниц:

- через `extension/options/` и background API
- экспортирует все интервалы и сводку товаров
- импорт поддерживает как интервалы, так и snapshot-подобные записи (конвертируются в интервалы)

### 7) Синк с сервером (двунаправленный)

- Расширение может работать полностью локально (`mode=local`).
- В `mode=sync`:
  - локальные обновления интервалов пушатся на сервер батчами;
  - серверные изменения подтягиваются обратно в локальную БД с merge.
- Синк запускается автоматически при старте расширения, при записи новых цен, а также при запросах минимумов/истории.
- Подтягивание с сервера селективное:
  - для карточек: только минимумы по запрошенному `pidKey[]` (`/api/min-batch`);
  - для истории товара: только история конкретного `pidKey` (`/api/history`).

## Быстрый старт

1. Расширение: загрузить папку `extension/`
2. (Опционально) сервер: из `server/` выполнить `python local_price_server.py`
